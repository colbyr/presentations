How does it work?

Initialize

Git is a command line application. git init creates a new repository in the current directory.

mkdir MyProject
cd MyProject
git init

Git is always watching...

Whenever you add, remove or change a file in your repository, git knows! You can check the status of the repo with git status which will return something like this:

# On branch master
#
# Initial commit
#
nothing to commit (create/copy files and use "git add" to track)

Track changes!

Before you can commit changes to the repository, you have to track them using git add ..

Running git status after adding changes will show that the changes have been "staged" for commit:

# On branch master
#
# Initial commit
#
# Changes to be committed:
#   (use "git rm --cached <file>..." to unstage)
#
#   new file:   readme.md
#

Commit

Once you've added any changes, you're ready to commit! Evey commit has a short message associated with it. The easiest way to specify a commit message is with the -m argument:

git commit -m 'first commit'

Produces:

[master (root-commit) 1d06ce0] first commit
 0 files changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 readme.md
